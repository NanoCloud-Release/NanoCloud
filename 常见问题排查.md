# 节点、运营商与故障排查

[返回项目首页](../README.md)

## 当前节点协议

节点协议包括 VLESS、TUIC 和 Hysteria。

## 运营商节点推荐

代理线路会受到省份、运营商、设备、IPv6、晚高峰和 QoS 策略影响，不存在全国所有网络环境下的绝对可用保证。

- 服务商的不同，推荐选择的节点不同：
  不同运营商节点推荐：
    移动用户：优先选择 韩国节点 或 新加坡节点。
    联通用户：建议选择 日本节点、新加坡节点 或 韩国节点。
    电信用户：可选择 BGP节点、香港节点 或 美国节点。

## 节点命名

- 节点命名：“通用”代表IPv4，宽带和流量都可用； IPv6“流量”，推荐使用流量。注意在手机上的开源代理软件需要打开IPv6，才能使用标注“流量”的节点，电脑也同理。

## 家庭 Wi-Fi 开启 IPv6

需要同时检查：

1. 光猫后台是否启用 IPv6。
2. 路由器后台是否启用 IPv6。
3. 宽带运营商是否向当前线路提供 IPv6。

如果不熟悉光猫或路由器配置，可联系运营商协助开通。部分地区、旧路由器和旧终端可能无法使用 IPv6，此时改用通用节点即可。

## 晚高峰与特殊网络

- 部分移动宽带可能缺少 IPv6，并在特定时段出现较明显的 QoS 波动。
- 部分基础电信宽带在晚高峰可能出现丢包。
- 农村或老旧家庭线路出现问题时，临时使用支持 IPv6 的流量卡可能比升级宽带更直接。

这些是经验性建议，不是全国统一测试结论。不同省份甚至同一城市的不同小区都可能有明显差异，最可靠的方法仍是在自己的网络中测试。

## 白天可用、晚上失效

如果节点每天在相近时段集中变红或断开，可能与本地运营商晚高峰或 QoS 有关。排查顺序：

1. 查看节点状态，确认不是服务端节点离线。
2. 切换通用节点。
3. 切换手机流量与家庭 Wi-Fi 交叉测试。
4. 测试另一家运营商网络。
5. 使用官方客户端确认是否能看到更多可用线路。

## 官网页面卡住

现象：

- 页面无法下滑
- 点击按钮没有反应
- 内容长时间不加载

处理：

1. 改用最新版 Chrome、Edge、Firefox 或 Safari。
2. 避免使用旧版内核或应用内置浏览器。
3. 如果夸克、Alook 等浏览器出现问题，先换浏览器再判断是否为网站故障。

## 客户端初始化失败
1. 彻底卸载旧客户端。
2. 安装最新版本。
3. 重新启动客户端。
4. 如果仍失败，在移动、联通、电信网络之间切换测试。

切换网络后恢复，通常说明问题更可能与本地运营商或当前网络配置有关。

## Windows 卸载重装后仍失败

可以使用可信的系统网络修复工具重置网络配置。
1. 打开火绒安全工具。
2. 找到“断网修复”。
3. 执行网络修复后重新启动客户端。
## 关机后开机无法联网

为避免系统代理设置残留：

1. 先在客户端中断开连接。
2. 从系统托盘彻底退出代理软件。
3. 再执行关机或重启。

## 节点全部变红

1. 打开控制台节点状态，确认是否为节点侧故障。
2. 从“流量”节点切换到“通用”节点。
3. 更换香港、日本、新加坡或美国方向测试。
4. 在手机流量和家庭 Wi-Fi 之间交叉测试。
5. 测试另一家运营商网络。
6. 使用官方客户端查看是否有更多线路。

## 仍无法解决

- 查阅 [NanoCloud 教程频道](https://t.me/NanoCloudWiki)。
- 使用 [小囡囡 Bot](https://t.me/NanoAir_bot)提交反馈。
- 在控制台创建工单，并提供设备系统、客户端版本、运营商、节点名称和错误现象。
