# 常见问题排查

适用对象：节点变红、家庭 Wi-Fi 不可用、晚高峰断流、客户端初始化失败或系统代理残留的用户。

最后核对：2026 年 7 月 15 日。

## 先做两分钟交叉测试

按顺序执行，恢复后就停止：

1. 打开控制台节点状态，确认不是服务端节点离线。
2. 从“流量”节点切换到“通用”节点。
3. 更换一个地区节点。
4. 在家庭 Wi-Fi 和手机流量之间交叉测试。
5. 更新订阅并彻底重启客户端。
6. 使用官方客户端确认是否能看到更多可用线路。

这组步骤可以先区分节点、IPv6、运营商、订阅和客户端问题。

## 先理解两种节点

- **通用节点**：使用 IPv4，家庭宽带和手机流量均可使用。
- **流量节点**：使用 IPv6，需要光猫、路由器、运营商、设备和客户端都支持并启用 IPv6。

节点协议可能包括 VLESS、TUIC 和 Hysteria。协议名称不能单独证明线路质量；判断时仍应以自己的运营商、时段和应用体验为准。

## 手机流量可用，家庭 Wi-Fi 不可用

先切换“通用”节点。如果通用节点可用，问题通常在家庭 IPv6 链路：

1. 检查光猫后台是否启用 IPv6。
2. 检查路由器后台是否启用 IPv6。
3. 检查宽带运营商是否向当前线路提供 IPv6。
4. 检查设备和代理客户端是否启用 IPv6。

不熟悉光猫或路由器配置时，可联系运营商协助。部分地区、旧路由器和旧终端无法完整支持 IPv6，此时继续使用“通用”节点即可。

## 白天可用，晚上失效

如果节点每天在相近时段集中变红或断开，可能与本地运营商晚高峰或 QoS 有关：

1. 记录城市、运营商、网络类型、客户端、时间和节点。
2. 在 20:00–23:00 持续观察超时、断流和重连，不只跑一次测速。
3. 切换“通用”节点和不同地区节点。
4. 用手机流量与家庭 Wi-Fi 交叉测试。
5. 有条件时测试另一家运营商。

经验性选择可作为起点：移动用户先试韩国或新加坡；联通用户先试日本、新加坡或韩国；电信用户可试 BGP、香港或美国。这不是全国统一保证。

## 节点全部变红

1. 查看控制台状态，确认是否为节点侧故障。
2. 更新订阅，并核对账户是否到期或流量用尽。
3. 从“流量”切到“通用”节点。
4. 更换香港、日本、新加坡或美国方向。
5. 在手机流量和家庭 Wi-Fi 之间交叉测试。
6. 使用官方客户端检查更多线路。

## 客户端初始化失败

1. 彻底退出并卸载旧客户端。
2. 从控制台提供的入口安装最新版本。
3. 重启设备后重新导入订阅。
4. 在移动、联通或电信网络之间切换测试。

切换网络后恢复，通常说明问题更可能与本地运营商或当前网络配置有关。

### Windows 卸载重装后仍失败

先确认系统代理没有残留，再使用 Windows 的“网络重置”功能。网络重置会移除并重新安装网络适配器，可能需要重新连接 Wi-Fi 或 VPN；执行前记录必要配置，完成后重启系统。

### 关机后开机无法联网

为减少系统代理设置残留：

1. 先在客户端中断开连接。
2. 从系统托盘彻底退出代理软件。
3. 再执行关机或重启。

## 官网页面卡住

现象可能包括页面无法下滑、点击按钮无反应或内容长时间不加载：

1. 改用最新版 Chrome、Edge、Firefox 或 Safari。
2. 避免旧版内核或应用内置浏览器。
3. 清除该站点缓存后重新打开。
4. 在另一网络中交叉测试。

## 仍无法解决

- 查阅 [NanoCloud 教程频道](https://t.me/NanoCloudWiki)。
- 使用 [小囡囡 Bot](https://t.me/NanoAir_bot) 提交反馈。
- 在控制台创建工单。
- 文档内容错误或过时时，提交 [GitHub Issue](https://github.com/NanoCloud-Release/NanoCloud/issues/new)。

反馈时提供：

- 设备系统与版本。
- 客户端名称与版本。
- 城市、运营商、家庭 Wi-Fi 或手机流量。
- 节点类型、地区与问题发生时间。
- 可复现步骤和完整错误文本。
