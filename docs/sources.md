部分论坛会拒绝匿名自动请求。自动链接检查返回 403 不一定代表浏览器用户无法访问，但这类来源的长期可达性较弱，关键结论应尽量补充一手公告。

| 内容 | 唯一完整版 | 其他入口如何处理 |
| --- | --- | --- |
| 行业事件、需求与成本比较 | 正式文档 | README 和产品主页只提供摘要与深链 |
| 套餐、客户端、节点和试用步骤 | 正式文档 | 产品主页显示核对日期与关键范围 |
| 故障排查 | 正式文档 | 根目录旧文件只保留迁移入口 |
| 产品定位、注册和 AFF 披露 | 产品主页为主 | README 与文档保留必要的短说明 |
| 仓库结构、变更和纠错 | GitHub | 产品主页与文档链接到具体入口 |

## 更新与编辑规则

1. 套餐和节点数据以控制台为准，并标注核对日期。
2. 外部服务状态查一手公告，不从搜索摘要直接下结论。
3. 单次测速不写成长期保证。
4. AFF 和普通注册链接同时公开。
5. 论坛内容标注来源类型与证据边界。
6. 已过时信息删除或明确标注时间。
7. 详细事实只在正式文档维护；README 和产品主页使用摘要与深链。
8. 发现错误时提交 [GitHub Issue](https://github.com/NanoCloud-Release/NanoCloud/issues/new) 或使用页面右上角的编辑入口。
