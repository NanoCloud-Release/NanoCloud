# 快速开始与 2 天试用

适用对象：已经决定试用 NanoCloud，需要完成首次连接并判断是否值得付费的用户。

最后核对：2026 年 7 月 15 日。

## 开始前准备

- 一台平时真正会使用的手机或电脑。
- 家庭宽带和手机流量，便于交叉判断运营商问题。
- 20:00–23:00 的晚高峰测试时间。
- 需要长期使用的 AI、流媒体和登录类网站。

## 第二步：完成首次连接

1. 从 NanoCloud 控制台提供的入口下载官方客户端，或使用控制台支持的一键导入。
2. 首次连接先选“通用”节点；它使用 IPv4，对家庭宽带和手机流量更通用。
3. 打开一个常用网页，确认流量确实经过代理。
4. 切换一个常用地区，再重启客户端，确认订阅更新和节点切换都能恢复。

控制台当前支持 Hiddify Next、Clash Verge、FlClash、Karing 和 sing-box；官方客户端支持 Android、Windows 和 macOS。更完整的客户端与套餐信息见 [NanoCloud 套餐与服务边界](nanocloud.md)。

### 什么时候使用“流量”节点

“流量”节点使用 IPv6。光猫、路由器、运营商、设备和代理客户端都需要启用 IPv6；任何一环不支持时，先改回“通用”节点。手机流量可用但家庭 Wi-Fi 不可用时，也应优先检查家庭 IPv6。

## 第三步：把试用用在真实任务上

不要用多次满速测试耗尽 5G。下面五项各完成一次即可：

| 测试 | 通过标准 | 需要记录 |
| --- | --- | --- |
| 晚高峰 | 20:00–23:00 能持续打开网页，没有频繁断流 | 城市、运营商、网络类型、节点 |
| AI 长对话 | 连续使用 10–20 分钟，模型路由和长回答正常 | 截断、重登、验证码、地区漂移 |
| 流媒体 | 1080p 连续播放并能拖动进度条 | 缓冲、解锁、切换节点后的恢复 |
| 登录类网站 | 常用账号没有反复验证或异常地区提醒 | 验证次数、识别国家、登录状态 |
| 故障恢复 | 更新订阅、切换节点或重启客户端后恢复 | 恢复步骤和耗时 |

单次测速只能说明当时的峰值，不代表持续吞吐、目标服务或未来可用性。

## 第四步：做购买决定

### 可以继续月付

- 关键任务在自己的网络中通过。
- 晚高峰没有不可接受的断流。
- 节点故障后有可用替代并能恢复。
- 共享出口、客户端和售后边界符合预期。

### 先不要购买

- 只有白天或手机流量可用，常用家庭网络无法稳定连接。
- 必须使用固定、独享或住宅出口 IP。
- 关键网站持续验证、地区漂移或账号异常。
- 需要企业 SLA、审计或明确赔付承诺。

陌生服务优先试用或月付。连续使用一段时间后再考虑更长周期；折扣只能降低价格，不能降低停运、维护、拔线和退款风险。
